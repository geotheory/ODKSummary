Troubleshooting


Windows connecting to domain

Windows machines can have difficulty accessing the local network domain that the Edison is serving (ie. http://mm/) while probably still being able to accesvie local network IP (ie. 192.168.0.1). If you get this problem the following procedure may resolve it:

          1. Go into Network Connections (I just typed "Network Connections" om the start icon -> run box)
          2. Right click on your Network Adapter and go into Properties
          3. Select "Internet Protocol Version 4 (TCP/IPv4)"
          4. Under the General Tab click Advanced
          5. Under the DNS Tab select the radio button that says "Append these DNS suffixes (in order)"
          6. Click "Addâ€¦"
          7. Enter a single period in the text box and click "Add."
          8. Click OK in the "Advanced TCP/IP Settings" Dialog
          9. Click OK in the "Local Area Connection Properties" Dialog
          10. Try connecting in browser (Chrome incognito mode, or Firefox private mode) to "mm/" or "http://mm/"

          source: http://jqnotes.blogspot.ca/2012/08/configure-windows-7-to-support-dnsmasq.html
